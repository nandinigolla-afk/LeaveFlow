# API Documentation

Full interactive documentation is auto-generated by springdoc-openapi and available once the backend is running:

- Swagger UI: `http://localhost:8080/swagger-ui.html` (local) or `https://<your-render-service>.onrender.com/swagger-ui.html` (deployed)
- Raw OpenAPI spec: `/v3/api-docs`

All endpoints are prefixed with `/api/v1`. All responses are wrapped in:

```json
{
  "success": true,
  "message": "Success",
  "data": { /* ... */ },
  "status": 200,
  "timestamp": "2026-07-14T10:30:00Z"
}
```

Paginated endpoints return `data` shaped as:

```json
{
  "content": [ /* items */ ],
  "page": 0,
  "size": 20,
  "totalElements": 42,
  "totalPages": 3,
  "last": false
}
```

Authenticated requests must include `Authorization: Bearer <accessToken>`.

## Auth — `/auth`

| Method | Path | Auth | Description |
|---|---|---|---|
| POST | `/auth/signup` | Public | Register a new employee account (role: EMPLOYEE) |
| POST | `/auth/login` | Public | Authenticate, returns access + refresh tokens |
| POST | `/auth/refresh` | Public | Exchange a refresh token for a new access/refresh pair |
| POST | `/auth/forgot-password` | Public | Request a password reset email (always 200, doesn't leak account existence) |
| POST | `/auth/reset-password` | Public | Reset password using a time-limited token |

## Error format

Non-2xx responses use the same `ApiResponse` envelope with `success: false`:

```json
{
  "success": false,
  "message": "Leave request not found with id: 42",
  "status": 404,
  "timestamp": "2026-07-14T10:30:00Z"
}
```

Validation errors (400) include a `data` map of field → message.
